## How to contribute

- Fork the repository and create your branch from main.
- Follow the existing code style and run linters/tests before submitting.
- Open a pull request describing the change and why it's needed.

## Issues

- For bug reports, include a clear description, steps to reproduce, and
  expected vs actual behavior.
- For feature requests, explain the problem and a suggested solution.

## Pull Requests

- Keep PRs small and focused.
- Include tests for new functionality where appropriate.
- Ensure the test suite passes locally.
- Mention any breaking changes in the PR description.

## Code Style

- Use consistent formatting. Prefer the project formatter (if any).
- Write clear, concise commit messages.
